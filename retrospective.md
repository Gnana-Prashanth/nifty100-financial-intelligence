# Sprint 1 Retrospective

## What Went Well

- Successfully built the ETL pipeline.
- Loaded all 12 datasets into SQLite.
- Implemented Data Quality checks and validation reports.
- Created `nifty100.db` and `load_audit.csv`.

## Challenges Faced

- Different headers in supporting datasets.
- Duplicate records causing `IntegrityError`s.
- Schema mismatches and missing columns.
- Foreign key and primary key issues.

## Key Learnings

- SQLite database design.
- ETL pipeline development.
- Data validation and quality checks.
- Debugging and problem-solving in real-world datasets.

## Overall

Sprint 1 was challenging but helped in understanding real-world ETL and database workflows. It provided hands-on experience in building an end-to-end ETL pipeline, handling data quality issues, and working with SQLite databases.
